# ビジネス系ポートフォリオサイト 構成案
（AIオペレーター / ECサイト運営 / AI業務改善）

作成日: 2026-07-02

## 1. 基本方針

### サイトは「1つ」に統合する
3方向それぞれ別サイトにするのではなく、**1つのサイトに3つの柱**として載せる。
理由:
- 採用担当者は「AIが使える × ECが分かる × 業務改善までできる」という**掛け合わせ**に価値を感じる。単体スキルの人材は多いが、複合人材は希少。
- イラストレーター用ポートフォリオとは**完全に分離**する（閲覧者・目的が違うため）。ただしフッターに「クリエイティブ実績はこちら」とリンクだけ置き、「クリエイティブも分かるAI人材」という差別化に使う。

### 最重要原則: 「作品を見せる」ではなく「成果を見せる」
イラストのポートフォリオは作品の質で勝負だが、ビジネス系は**数値とBefore/After**で勝負。
「AI秘書を作りました」ではなく「予定通知を自動化し、確認作業を1日◯分削減」と書く。

## 2. ページ構成（推奨: 1ページ完結型 + 実績詳細ページ）

```
トップページ（縦スクロール1枚）
├─ ① ヒーロー: キャッチコピー + 肩書き
├─ ② できること3本柱（AIオペレーター / EC運営 / 業務改善）
├─ ③ 実績カード一覧（数値付き）→ クリックで詳細ページへ
├─ ④ 使用ツール・スキル一覧
├─ ⑤ 仕事の進め方（ヒアリング→設計→構築→運用）
├─ ⑥ プロフィール
└─ ⑦ お問い合わせ
実績詳細ページ × 4〜6件
```

### ① ヒーロー
- 例: 「AIで"手作業"をなくす。― AI活用 × EC運営 × 業務自動化」
- デザインはイラスト版と違い、**白基調・信頼感重視**（コーポレート寄り）。カラフルにしない。

### ② 3本柱セクション
| 柱 | 見せ方 |
|---|---|
| AIオペレーター | Claude / ChatGPT / Gemini を使い分けた実務経験。プロンプト設計、AIツール選定 |
| ECサイト運営 | 商品登録・受注対応・在庫/売上管理など運営業務の経験（あるものを具体的に） |
| AI業務改善 | 「非エンジニアの現場にAI自動化を導入できる」ことを実例で証明 |

### ③ 実績カード（3本柱ごとにカテゴリ分けして表示）

実績が10件超あるため、フラットに並べず**タブ or フィルタ付きのカード一覧**にする。
「代表実績3件」をカテゴリ横断でトップに固定し、残りはカテゴリ内に格納。

**A. AI業務自動化・効率化**
1. **AI秘書（Claude版）** — Google Calendar × Google Chat 連携の自動通知。重複チェック付き
2. **AI秘書（GAS × ChatGPT版）** — サーバーレスで動く第2世代。異なるAI基盤での再構築力の証明
3. **抽選ハガキOCRツール** — GPT Vision で手書きハガキ→Excel化 + 過去当選者の重複検出
4. **ストレスチェック集計自動化** — formrun回答→厚労省版プログラムへのCSV取込
5. **社内チャットボットアプリ** — 社内問い合わせ対応の自動化

**B. データ収集・見える化**
6. **複数店舗情報のリスト化ツール** — WEBサイトから約200店舗の情報を自動収集・リスト化
7. **ダッシュボードアプリ** — データの見える化。経営判断・現場共有に使える形へ
8. **従業員シフト表アプリ** — シフト管理の効率化
9. **QGISエリアマップ作成** — 配布エリア等の地図データ化・ポリゴン作成

**C. クリエイティブ × 自動化（差別化の切り札）**
10. **Illustrator / Photoshop 作業の自動化** — スクリプトによる定型作業の時短。デザイン実務が分かるからこそ組める自動化
11. **microCMSでの効率化** — コンテンツ管理・更新フローの改善
12. **ひらがな学習ゲーム「はなさかわんわん」** — 企画〜実装をAI協働で完遂した例

（ECの運営実績があれば「D. EC運営」として追加: 運営店舗・改善した指標など）

使用環境について: Claude Code / Antigravity など複数のAI開発環境を実務で使い分けている事実は
「特定ツール依存ではない」証明になるので、各実績カードに使用環境を小さく明記する。

各詳細ページのフォーマットを統一する:
- **課題**（Before）→ **やったこと**（使用ツール・構成図）→ **成果**（After・数値）
- スクリーンショット or 30秒デモ動画を必ず1点入れる
- 例: 店舗リスト化「手作業なら200店×◯分=◯時間 → 自動化で◯分」のような換算を必ず入れる

### ④ スキル・ツール
- AI: Claude / Claude Code / ChatGPT (GPT Vision) / Gemini / Antigravity / プロンプト設計
- 自動化: Google Apps Script / Google Workspace連携 / Windowsタスク自動化 / Illustrator・Photoshopスクリプト
- データ・地図: ダッシュボード構築 / スクレイピング・リスト化 / QGIS
- Web・CMS: HTML/CSS / GitHub Pages / microCMS / formrun / Excel

### ⑤ 仕事の進め方
「ヒアリング → 課題整理 → ツール選定 → 構築 → 運用・改善」の流れを図で。
採用側の不安（丸投げして大丈夫か）を消すセクション。

## 3. 採用につながるためのポイント（分析）

1. **数値がすべて**。時間削減・工数削減・ミス削減を必ず書く。正確な計測がなければ「推定◯時間/月」でよい。
2. **再現性を見せる**。「あなたの会社でも同じことができます」と伝わる書き方。AI秘書を2つの基盤（Claude版/GAS+ChatGPT版）で作った事実は再現性の強力な証拠。
3. **動くものを見せる**。GitHub Pagesでデモ公開 or 動画。「本当に動くのか」の疑念を消す。
4. **非エンジニア目線を武器に**。「現場の人が使える形に落とし込める」ことはエンジニアより刺さる場面が多い。
5. **問い合わせ導線を短く**。フォーム or メールリンクをヘッダーとフッター両方に。

## 4. 公開方法
**GitHub Pages（無料）に決定**（イラスト版と同じ方式、別リポジトリで分ける）。
